# FHIR-MCP v0.2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Production-code tasks follow strict TDD per the `test-driven-development` skill: red, verify red, green, verify green, then commit.

**Goal:** Ship `fhir-mcp` v0.2.0, a minor release that (a) plugs a trained classifier into the existing `Router` seam via a new `ClassifierRouter` opt-in, (b) replaces ad hoc logging with `structlog` and exposes a `/health` liveness route on the SSE app, and (c) cuts the 0.2.0 PyPI release and bumps the MCP registry entry.

**Architecture:** v0.2.0 is additive over v0.1.0. The `Router` protocol in `src/fhir_mcp/adherence/routing.py` is the single seam; v0.2 adds one new implementation (`ClassifierRouter`) selected at process start by the env var `FHIR_MCP_ROUTER` (default `heuristic`). The classifier artifact (weights and inference code) is loaded from a path or URL via a `ClassifierLoader` interface; the actual artifact is TBD per `CLAUDE.md` and is gated behind Pre-flight Open Question 1. Structured logging is a process-wide change in `src/fhir_mcp/logging_config.py` consumed by `__main__.py` and `server.py`. The `/health` route is one extra `Route` in `FhirMcpServer.sse_app()`.

**Tech Stack:** Python 3.11+, existing `mcp`, `fhir.resources`, `pydantic`, `pytest`, `pytest-asyncio`, `ruff`, `mypy`, `httpx` (already a `proxy` extra, reused for classifier-fetch tests), plus new dep `structlog>=24.0`. Build, CI, and Docker stack from v0.1.0 unchanged.

## Pre-flight: Open questions

These must be resolved before Phase 1 Task 1.1 starts. Defaults are listed; user override sets the actual value before the task begins.

1. **Classifier artifact location.** Default: a local directory `src/fhir_mcp/adherence/classifier_weights/` checked into the repo, containing a `manifest.json` plus model files. Override option: an HTTP(S) URL that `ClassifierLoader` fetches at startup and caches under `$XDG_CACHE_HOME/fhir-mcp/classifier/`. The user must confirm one of these two before Task 1.1.
2. **Classifier interface contract.** Default: a callable `predict(question: str) -> tuple[Literal["structured", "narrative"], float]` returning `(label, probability)` where probability is the model's confidence in the chosen label, in `[0.0, 1.0]`. Override: confirm or amend on artifact delivery. Until confirmed, Phase 1 uses this exact contract in tests via a mock.
3. **Backward compatibility.** Default: env var `FHIR_MCP_ROUTER` accepts `heuristic` (default) or `classifier`. Unset or unknown values fall back to `heuristic` with a single warning log line. v0.2 is strictly additive; v0.1.x deployments keep working without setting the var.
4. **`structlog` version pin.** Default: `structlog>=24.0`. Override only if a known compatibility issue forces an older line.
5. **Confidence threshold for ambiguous fallback.** Default: if the classifier returns probability `< 0.60`, `ClassifierRouter.detect` returns `Intent.AMBIGUOUS` so `compute_adherence` runs both branches. This mirrors the v0.1 heuristic ambiguous semantics. Override: a different threshold or no threshold (always trust the classifier).

If Pre-flight 1 cannot be resolved within 5 business days of starting v0.2.0, escalate to the user and consider deferring Phase 1 to v0.2.1, shipping Phases 2 and 3 alone as v0.2.0. Phases 2 and 3 do not depend on Phase 1.

  from typing import Callable, Literal

  ClassifierPrediction = tuple[Literal["structured", "narrative"], float]
  ClassifierCallable = Callable[[str], ClassifierPrediction]


  class ClassifierRouter:
      """Router backed by a trained classifier (Jani et al., May 2026).

      The classifier is supplied as a callable matching ClassifierCallable.
      Confidence below `threshold` returns Intent.AMBIGUOUS so
      compute_adherence runs both branches.
      """

      def __init__(
          self,
          predict: ClassifierCallable,
          threshold: float = 0.60,
      ) -> None:
          self._predict = predict
          self._threshold = threshold

      def detect(self, question: str) -> Intent:
          label, probability = self._predict(question)
          if probability < self._threshold:
              return Intent.AMBIGUOUS
          if label == "structured":
              return Intent.STRUCTURED
          if label == "narrative":
              return Intent.NARRATIVE
          return Intent.AMBIGUOUS
  ```

  from __future__ import annotations

  import json
  from pathlib import Path
  from typing import Callable, Literal

  ClassifierPrediction = tuple[Literal["structured", "narrative"], float]
  ClassifierCallable = Callable[[str], ClassifierPrediction]


  class ArtifactNotFoundError(FileNotFoundError):
      """Raised when the classifier artifact directory is missing files."""


  _REQUIRED_LABELS = {"structured", "narrative"}


  def load_classifier_from_path(path: Path) -> ClassifierCallable:
      """Load a classifier artifact from a local directory.

      The directory must contain a manifest.json with at least the keys
      'model', 'version', and 'labels'. The 'labels' list must equal
      ['structured', 'narrative'] in either order.

      Returns a callable conforming to ClassifierCallable. Until the real
      artifact ships (see Pre-flight Open Question 1 in
      docs/superpowers/plans/2026-05-18-fhir-mcp-v0.2.0.md), the loader
      returns a deterministic stub keyed off the manifest version so tests
      can exercise the seam without weights.
      """
      manifest_path = path / "manifest.json"
      if not manifest_path.exists():
          raise ArtifactNotFoundError(
              f"manifest.json not found in classifier artifact: {path}"
          )
      manifest = json.loads(manifest_path.read_text())
      labels = set(manifest.get("labels", []))
      if labels != _REQUIRED_LABELS:
          raise ValueError(
              f"manifest labels must be exactly {_REQUIRED_LABELS}, got {labels}"
          )

      def predict(question: str) -> ClassifierPrediction:
          # Stub: real inference replaces this body when the artifact lands.
          q = question.lower()
          structured_terms = ("did", "take", "refill", "how many", "between")
          narrative_terms = ("describe", "overall", "pattern", "summari")
          if any(t in q for t in structured_terms):
              return ("structured", 0.95)
          if any(t in q for t in narrative_terms):
              return ("narrative", 0.90)
          return ("structured", 0.55)

      return predict
  ```

  ```python
  import logging
  import os
  from pathlib import Path

  _logger = logging.getLogger(__name__)

  _DEFAULT_CLASSIFIER_PATH = (
      Path(__file__).parent / "classifier_weights"
  )


  def build_router_from_env() -> Router:
      """Pick a Router implementation from environment variables.

      FHIR_MCP_ROUTER:
          'heuristic' (default) returns HeuristicRouter.
          'classifier' returns ClassifierRouter loaded from the path in
          FHIR_MCP_CLASSIFIER_PATH (defaults to
          src/fhir_mcp/adherence/classifier_weights/).
          Any other value logs a warning and returns HeuristicRouter.
      """
      choice = os.environ.get("FHIR_MCP_ROUTER", "heuristic").strip().lower()
      if choice == "heuristic":
          return HeuristicRouter()
      if choice == "classifier":
          from fhir_mcp.adherence.classifier_loader import (
              ArtifactNotFoundError,
              load_classifier_from_path,
          )

          artifact_path = Path(
              os.environ.get(
                  "FHIR_MCP_CLASSIFIER_PATH", str(_DEFAULT_CLASSIFIER_PATH)
              )
          )
          try:
              predict = load_classifier_from_path(artifact_path)
          except (ArtifactNotFoundError, ValueError) as exc:
              _logger.warning(
                  "classifier artifact unavailable at %s (%s); "
                  "falling back to HeuristicRouter",
                  artifact_path,
                  exc,
              )
              return HeuristicRouter()
          return ClassifierRouter(predict=predict)
      _logger.warning(
          "unknown FHIR_MCP_ROUTER value %r; falling back to HeuristicRouter",
          choice,
      )
      return HeuristicRouter()
  ```

  ```python
  from __future__ import annotations

  import logging
  import sys
  from typing import Literal

  import structlog

  LogFormat = Literal["json", "console"]

  _VALID_FORMATS: tuple[LogFormat, ...] = ("json", "console")


  def configure_logging(log_format: str = "console", level: str = "INFO") -> None:
      """Configure structlog and stdlib logging for the whole process.

      Call once at process start (in __main__.main) before any logger is used.

      Args:
          log_format: 'json' for newline-delimited JSON to stdout
              (Oracle / container friendly), 'console' for a human-readable
              colorless format.
          level: standard logging level name ('DEBUG', 'INFO', ...).
      """
      if log_format not in _VALID_FORMATS:
          raise ValueError(
              f"log_format must be one of {_VALID_FORMATS}, got {log_format!r}"
          )

      level_int = getattr(logging, level.upper(), logging.INFO)

      shared_processors: list[structlog.types.Processor] = [
          structlog.contextvars.merge_contextvars,
          structlog.processors.add_log_level,
          structlog.processors.TimeStamper(fmt="iso", utc=True),
      ]

      if log_format == "json":
          renderer: structlog.types.Processor = structlog.processors.JSONRenderer()
      else:
          renderer = structlog.dev.ConsoleRenderer(colors=False)

      formatter = structlog.stdlib.ProcessorFormatter(
          foreign_pre_chain=shared_processors,
          processors=[
              structlog.stdlib.ProcessorFormatter.remove_processors_meta,
              renderer,
          ],
      )

      handler = logging.StreamHandler(stream=sys.stdout)
      handler.setFormatter(formatter)

      root = logging.getLogger()
      root.handlers = [handler]
      root.setLevel(level_int)

      structlog.configure(
          processors=[
              *shared_processors,
              structlog.stdlib.ProcessorFormatter.wrap_for_formatter,
          ],
          logger_factory=structlog.stdlib.LoggerFactory(),
          cache_logger_on_first_use=True,
      )
  ```

      from mcp.server.sse import SseServerTransport
      from starlette.applications import Starlette
      from starlette.responses import JSONResponse
      from starlette.routing import Mount, Route

      from fhir_mcp import __version__

      transport = SseServerTransport("/messages/")
      mcp_server = self._mcp

              Route("/sse", endpoint=handle_sse),
              Mount("/messages", app=transport.handle_post_message),
          ]
      )

## Non-goals for v0.2.0

- New FHIR resource tools. The five v0.1.0 tools are the surface for v0.2.
- AuthN / authZ on the SSE endpoint. `/health` is intentionally unauthenticated; the rest of the SSE app inherits the v0.1.0 deployment posture (read-only synthetic data).
- Real-PHI handling. Same scope rule as v0.1.0.
- Metrics endpoint (`/metrics`, Prometheus). Liveness only in v0.2.0; readiness and metrics are v0.3 candidates.
- Replacing `HeuristicRouter`. It remains the default for v0.2.x; switching defaults is a v0.3 conversation.
- Training the classifier. v0.2.0 only consumes the artifact; training lives in the preprint authors' repo per `CLAUDE.md`.
- Distributing classifier weights inside the PyPI wheel. Weights ship via the path or URL in `FHIR_MCP_CLASSIFIER_PATH`; the wheel only ships the loader and an empty `classifier_weights/` directory.
- HTTP-URL fetch path in `classifier_loader`. Default in Pre-flight Open Question 1 is the local-directory path; if the user picks the URL override, that path lands in v0.2.1 along with the cache directory and integrity check.
